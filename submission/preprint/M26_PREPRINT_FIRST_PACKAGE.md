## Purpose of M26

M26 prepares the project for a preprint-first submission route.

This stage does not submit the manuscript to a journal or preprint server. It organizes the already prepared files into a route suitable for public scholarly submission.

## Recommended first submission framing

The recommended first framing is:

**A Lean 4 / Mathlib supported formal-methods artifact for verified milestone kernels in boundary-threshold reasoning.**

The submission should emphasize:

- public GitHub repository;
- passing Lean CI;
- Zenodo DOI;
- reproducibility files;
- audit logs;
- checksum records;
- bounded formal-verification claim;
- distinction between verified kernels and broader monograph exposition.
