## Recommended submission path

The recommended path is:

1. Use the GitHub repository and Zenodo DOI as the public verification record.
2. Prepare the manuscript as a bounded formal-methods / mathematical-software submission.
3. Emphasize the verified Lean milestone chain and audit evidence.
4. Avoid claiming complete mechanization of the entire monograph.
5. Submit only after the abstract, introduction, claim boundary, and reproducibility section are aligned.

## Submission category

Recommended category:

- formal verification support artifact;
- Lean 4 / Mathlib mathematical software artifact;
- boundary-threshold reasoning framework with verified milestone kernels.

## Not recommended as first framing

Avoid framing this as:

- a replacement for ZFC, HoTT, topology, probability, measure theory, category theory, or optimization;
- a fully mechanized monograph;
- a complete new foundation for all mathematics;
- a universal solver.

## Final decision

M22 prepares the project for submission, but does not itself submit the manuscript.

The next stage is M23: final manuscript revision and venue-specific formatting.
