# BEC v2.0 M15 Publication / Reproducibility Package

## Purpose

This package is the publication-level reproducibility wrapper for the BEC v2.0 Lean-checked milestone chain.

It collects the available milestone evidence packages M1-M14, provides a single release ledger, records SHA256 checksums, and states the formal claim boundary for public release.

## Reproducibility workflow

1. Extract this package.
2. Copy a milestone package into a Lean/Lake Mathlib project root.
3. Extract the milestone package.
4. Run its included `check_becv2_*.sh` script.
5. Compare the generated pass log and SHA256 sums with the ledger.

For example:

```bash
cd ~/URDM_v0_5_3_Mathlib_Minimal_Theorem_Kernel_v0_1
unzip -o BECv2_M14_Categorical_Lean431_Mathlib_PASS.zip
chmod +x check_becv2_m14.sh
./check_becv2_m14.sh
```

Expected pattern:

```text
Compile check
Blocked-token scan
PASS: blocked-token scan clean.
PASS: <milestone file>.lean compiled successfully.
```

## Claim boundary

The verified claim is:

> BEC v2.0 has a Lean 4.31.0 / Mathlib checked milestone chain M1-M14, including separate verified kernels, audit bundles, monograph integration drafts, and integrated all-in-one kernels.

The following stronger claims are not made:

- the entire monograph is fully mechanized;
- every external solver backend is end-to-end verified;
- all stochastic-process analysis is fully formalized in Mathlib;
- BEC replaces ZFC, HoTT, topology, measure theory, probability theory, or category theory;
- BEC is an unrestricted universal solver.
